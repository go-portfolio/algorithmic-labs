# Что делается в данной программе ``dynamic_pool_worker.go``?
Происходит регулировка количества воркеров по количеств входящих задач. Если задач мало и 
задачи не выполнялись в течение 3-х секунд то воркер умрёт. Если задач много то добавляется 
новый воркер. То есть с таким воркером можно быть увереным всегда что будет достаточное количество воркеров
и не будет слишком много воркеров для задачи. Программа может применяться в высоконагруженных системах,
когда при пике нагрузки количество воркеров сразу возрастает, или при слабой нагрузке воркеры уничтожаются.
Воркеры создаются в функции. Слежка за воркерами происходит в горутине. Основными моментами,
где происходит регулировка количества воркеров  - это time.After в select/case, когда при простаивании
select в течение 3-х секунд воркер уничтожается. И горутина в функции monitor постоянно отслеживает 
количество воркеров и при недостатке воркеров сразу же их добавляет
